# Flux Linux v1.5 Release Notes

### 1. KDE Plasma 6 Desktop Environment
Full KDE Plasma 6 support has been added as a selectable desktop environment:
- **Full Suite**: Switched to `kde-standard` metapackage, ensuring all core KDE apps are installed — System Settings, Dolphin, Konsole, Spectacle, Kate, and more.
- **Launch/Stop from Home Screen**: KDE (and XFCE4) launch and stop buttons are now on the home screen distro card instead of the settings page.
- **Smart Stop**: The Stop button correctly identifies the running desktop environment (XFCE4 or KDE) and terminates only that session.
- **Re-run Options**: Settings page now includes options to re-run XFCE4 Desktop, XFCE4 Customization, KDE Plasma Desktop, and KDE Customization independently.

### 2. KDE Bug Fixes
- **Window Decorations Fixed**: Disabled OpenGL compositing in `kwinrc` (`Enabled=false`, `Backend=QPainter`). KWin was failing to initialize OpenGL in proot, which stripped window decorations as a side effect.
- **Customization Script Fixed**: Removed `qt5-style-plugins` from the critical apt install block (package not available on Debian Trixie) — preventing the customization script from failing.
- **Stop Script Fixed**: KDE stop now uses correct Plasma 6 process names (`kded6`, `plasma_session`, `startplasma-x11`) via the same `proot-distro login + killall` approach used by the working XFCE4 stop.
- **PulseAudio Integration**: Audio now properly initialised in KDE sessions — PA started from Termux side before entering proot, connected via `PULSE_SERVER=tcp:127.0.0.1` (same pattern as XFCE4).
- **Error Handling**: KDE installation and customization scripts now log to `/tmp/fluxlinux_kde_*.log` and notify the app of success/failure via `am start` callbacks.

### 3. Zsh Shell Startup Optimizations
Shell startup time reduced massively (from 35+ seconds to ~1.5 seconds on PRoot):
- **Removed `zsh-autocomplete`**: This plugin was the primary cause of slow startup on PRoot systems due to high system call frequency.
- **Backgrounded Visuals**: `fastfetch` and `pokemon-colorscripts` now run asynchronously (`{ ... } &!`) — the shell prompt is immediately ready.
- **Disabled Update Prompts**: `DISABLE_UPDATE_PROMPT=true` and `DISABLE_AUTO_UPDATE=true` prevent oh-my-zsh from checking for updates on every launch.
- **Faster Init**: `ZSH_DISABLE_COMPFIX=true` skips `compaudit` on startup.
